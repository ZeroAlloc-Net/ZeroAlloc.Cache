# ZC0001 — Sliding TTL Ignored by HybridCache L2

**Severity:** Warning

`Sliding = true` was combined with `UseHybridCache = true`. HybridCache's distributed (L2) tier uses absolute TTL only; the sliding window is applied to the in-process (L1) tier but is silently ignored for the distributed tier. This means cache entries in the distributed store expire at the original absolute deadline regardless of access frequency, which is almost certainly not the intended behavior.

---

## Example

```csharp
// ZC0001: Sliding = true has no effect on the HybridCache L2 (distributed) tier
[Cache(TtlMs = 60_000, Sliding = true, UseHybridCache = true)]
public interface IProductRepository
{
    ValueTask<Product?> GetByIdAsync(int id, CancellationToken ct);
}
```

---

## How to fix

**Option A — Remove `Sliding = true`** if distributed caching is the goal and absolute TTL is acceptable:

```csharp
[Cache(TtlMs = 60_000, UseHybridCache = true)]
public interface IProductRepository
{
    ValueTask<Product?> GetByIdAsync(int id, CancellationToken ct);
}
```

**Option B — Remove `UseHybridCache = true`** if sliding expiration is important and in-process caching is sufficient:

```csharp
[Cache(TtlMs = 60_000, Sliding = true)]
public interface IProductRepository
{
    ValueTask<Product?> GetByIdAsync(int id, CancellationToken ct);
}
```

---

## When to suppress

Never. The combination of `Sliding = true` and `UseHybridCache = true` always produces inconsistent expiration behavior between the L1 and L2 tiers. Suppressing the warning does not fix the underlying behavioral discrepancy.
